Programa:

* Foram utilizados 8000 números afim de maximizar o tempo de processamento,
  pois utilizando 1000 números a execução se torna tão rápida que não é
  possível detectar alguma diferença entre os tempos antes da execução e após,
  para determinar o tempo de execução do método.

* O combinado de 10000 números não se torna possível de executar no modo DOS
  utilizando a estrutura ARQ sugerida. A maior quantidade de números possível
  encontrada por nós foi 8000.

* Para compilar o programa, deve-se utilizar o TargetExpress e fazer as modificações:
  - Platform: DOS (Overlay)
  - Math Support: Floating Point
  - Libraries: BGI

* Caso dê algum problema de compilação, enviamos junto o arquivo executável
  já compilado.

* Foram inclusas as seguintes linhas para a compatibilidade do floating point no modo
  16-bit:
extern _floatconvert;
#pragma extref _floatconvert


Pacote Gráfico:

* As funções para relatorio estatístico e gráfico foram otimizadas para funcionarem com
  qualquer número de métodos, definido pela constante METODOS.

* Para gerar o fator de escala, foi criada a função GeraFatorEscala, que, utilizando
  iterações sucessivas, determina um fator de escala ótimo pra caber em 300 pixels de
  altura da tela do modo gráfico.

* Ocorre um problema de impressão indevida de lixo de memória no modo gráfico. Não 
  encontramos uma maneira de lidar com isso, então mandamos escrever o lixo de memória
  para fora da tela.
